# PWA Location Notes
**Прогрессивное веб-приложение (PWA)** для создания текстовых заметок с автоматическим сохранением географических координат.  
Все данные хранятся локально в браузере (IndexedDB), приложение работает офлайн и может быть установлено на мобильное устройство как нативное.
## Цель проекта
Разработать полнофункциональное PWA, демонстрирующее:
- использование современных веб-технологий (Service Worker, Web App Manifest, Geolocation API, IndexedDB);
- работу в офлайн-режиме;
- кроссплатформенность (десктоп + мобильные устройства);
- интуитивный интерфейс для ведения заметок с геопозицией.
## Функциональные возможности
- ✍️ **Создание заметок** – введите текст в поле и нажмите «Добавить».
- 📍 **Автоматическая геолокация** – при создании заметки браузер запрашивает доступ к местоположению; координаты (широта, долгота) сохраняются вместе с заметкой.
- 💾 **Локальное хранение** – все заметки хранятся в IndexedDB, данные не покидают ваше устройство.
- 🗑️ **Удаление заметок** – каждая заметка имеет кнопку удаления.
- 📱 **Установка как PWA** – после установки на домашний экран смартфона приложение запускается полноэкранно, без адресной строки браузера.
- 🌐 **Офлайн-доступ** – после первого открытия приложение полностью работает без интернета (Service Worker кэширует все ресурсы).
- 📱 **Адаптивный дизайн** – интерфейс корректно отображается на экранах любых размеров (десктоп, планшет, телефон).
## Используемые технологии
 Технология 	 Назначение 
HTML5 + CSS3	- Структура и стилизация интерфейса.
Vanilla JavaScript (ES6)	- Вся клиентская логика без внешних библиотек. 
IndexedDB	- Встроенная в браузер документо-ориентированная база данных для хранения заметок. 
Geolocation API	- Определение текущих координат устройства. 
Service Worker (sw.js)	- Кэширование ресурсов и реализация офлайн-режима. 
Web App Manifest (manifest.json)	- Настройка внешнего вида приложения при установке. 
http-server / localtunnel	- Инструменты для локального запуска и публичного доступа. 
    Требования для запуска
- Браузер: любой современный (Chrome 60+, Edge 80+, Firefox 60+, Safari 14+, Yandex Browser) с поддержкой Service Worker и IndexedDB.
- Веб-сервер: локальный сервер (например, `http-server`, `Python http.server`, Live Server в VS Code) – для корректной работы PWA требуется протокол `http://` или `https://` (не `file://`).
- Node.js (опционально) – для установки `http-server` и `localtunnel` через `npx`.
Инструкция по установке и запуску
1. Клонирование репозитория
```bash
git clone https://github.com/konstantinfedoseew-ai/PWA_location.git
cd PWA_location
2. Запуск локального веб-сервера
Выберите один из способов:

Способ А – с помощью http-server (рекомендуется)
Убедитесь, что у вас установлен Node.js (можно проверить командой node -v). Затем выполните:

bash
npx http-server -a 0.0.0.0 -p 8080 -c-1
-a 0.0.0.0 – разрешает доступ с других устройств в локальной сети.
-p 8080 – порт (можно изменить на другой, например, 3000).

-c-1 – отключает кэширование на стороне сервера (удобно для разработки).

Способ Б – с помощью Python (если нет Node.js)
Python 3.x:

bash
python -m http.server 8080 --bind 0.0.0.0
Python 2.x:

bash
python -m SimpleHTTPServer 8080
Способ В – с помощью VS Code Live Server
Если вы используете VS Code, установите расширение Live Server, откройте папку проекта и нажмите Go Live в правом нижнем углу (по умолчанию сервер запускается на порту 5500, но доступен только локально, без привязки к 0.0.0.0).

3. Открытие приложения в браузере
На компьютере: перейдите по адресу http://localhost:8080 (или используйте ваш порт).

На мобильном устройстве в той же Wi-Fi сети: узнайте IP-адрес компьютера:
Windows: ipconfig (в командной строке), найдите IPv4-адрес.
Затем на телефоне откройте http://<IP-вашего-компьютера>:8080.

4. Доступ через интернет (если телефон не в локальной сети)
Используйте localtunnel для создания временного публичного URL:

bash
npx localtunnel --port 8080
Скопируйте полученную ссылку (например, https://xxxx.loca.lt) и откройте её на любом устройстве с интернетом.

Установка как PWA
Откройте приложение в браузере на мобильном устройстве.
В Chrome / Yandex Browser / Samsung Internet:
Нажмите на кнопку меню (три точки).
Выберите «На экран «Домой»» или «Установить приложение».
Следуйте инструкциям на экране - приложение появится среди установленных программ с собственным значком.
После установки запуск происходит полноэкранно, без интерфейса браузера.
Работа с приложением
Просмотр заметок – при открытии отображаются все сохранённые заметки в виде списка.
Создание заметки:
Введите текст в текстовое поле и нажмите кнопку «Добавить».
Приложение запросит разрешение на доступ к геолокации (если ещё не было). Разрешите – и заметка сохранится вместе с текущими координатами.
Если разрешение не дано или координаты не определяются (например, GPS выключен), заметка сохранится без координат.
Удаление заметки – под каждой заметкой есть кнопка «Удалить»; после подтверждения заметка исчезает из базы данных.

Структура записи:

json
{
  "id": "уникальный_автоинкремент",
  "text": "Текст заметки",
  "latitude": 55.7558,   // широта (число) или null
  "longitude": 37.6173,  // долгота (число) или null
  "timestamp": 1625097600000 // время создания (Unix timestamp)
}
Все операции (добавление, чтение, удаление) выполняются асинхронно с использованием Promise.

Service Worker
Файл sw.js кэширует все статические ресурсы (index.html, style.css, app.js, manifest.json, иконки) при первом заходе.

Геолокация
Используется navigator.geolocation.getCurrentPosition().

Запрос разрешения происходит только при первом создании заметки (если разрешение ещё не было дано/отклонено).
